# 질의 프롬프트 구조 — 건축법규 RAG 시스템

## 전체 흐름 개요

```
사용자 질의
    │
    ▼
[Pass 1] 쟁점 식별 · 검색 파라미터 생성
    │
    │  → law_hints, definition_terms, relation_types, question_type
    ▼
[검색 (05_Retriever.py)]
    │  ① 직접 조문 페칭 (exact: law_hints 명시 조문)
    │  ② 하이브리드 검색 (벡터 + BM25 → RRF 병합)
    │  ③ 질의회신 선례 검색 (qa_precedents)
    │  ④ 판례 검색 (court_cases)
    │  ⑤ 메모 주입 (fetch_linked_memos — 결정론적)
    ▼
[Pass 2] 최종 답변 생성
    │  컨텍스트: [직접 참조 조문] + [관련 법령 조문] + [선례] + [판례] + [메모]
    ▼
답변 반환
```

## Pass 1 — 쟁점 식별 프롬프트

**역할**: 질의를 분석하여 검색에 필요한 파라미터를 추출. 최종 답변은 생성하지 않음.

### 출력 섹션 (순서대로)

#### [쟁점 식별]
- 핵심 쟁점 1~2문장
- 질문 유형: `단일조문형` | `복수조문탐색형` | `조건분기형`

| 유형 | 설명 |
|------|------|
| 단일조문형 | 특정 조문 하나로 해결 가능 |
| 복수조문탐색형 | 여러 조문 검토 필요 |
| 조건분기형 | 조건(면적·층수·용도 등)에 따라 결론이 달라짐 |

#### [검색 트리거]
직접 적용될 조문/키워드 나열. 사유 태그 포함:
`정의참조` | `요건위임` | `의제준용` | `적용배제` | `용도시설분류`

#### [정의 확인 용어]
쟁점 해결 전 법령상 정의를 먼저 확인해야 하는 핵심 용어.
- 탐색 우선순위: 질의 법령 내 정의 조문 먼저 → 없으면 타 법령

#### [판례-법령 관계 유형 분류]
복수 유형 허용. weight 부여 (0.5 미만 생략).

| 코드 | 이름 | 질문의 성격 |
|------|------|-------------|
| DEF_EXP | 정의확장형 | "~에 ~도 포함되나요?" |
| SCOPE_CL | 적용범위확정형 | "~에도 ~이 적용되나요?" |
| REQ_INT | 요건해석형 | "~의 요건은?" |
| EXCEPT | 예외인정형 | "~임에도 불구하고 가능한가요?" |
| INTER_ART | 조문간관계 해석형 | "어느 조문이 우선?" |
| PROC_DISC | 절차·재량 확인형 | "재량 범위는?" |
| SANC_SC | 벌칙·제재 범위형 | "위반 시 제재는?" |

#### [구조화 데이터] (JSON — 파싱 필수)
```json
{
  "question_type": "단일조문형 | 복수조문탐색형 | 조건분기형",
  "law_hints": ["건축법 제19조", "건축법 시행령 별표1"],
  "definition_terms": ["대수선", "저당권등"],
  "relation_types": [
    {"type": "SCOPE_CL", "reason": "이유 1문장", "weight": 1.0},
    {"type": "REQ_INT",  "reason": "이유 1문장", "weight": 0.7}
  ]
}
```

## 검색 단계 — 컨텍스트 구성 (05_Retriever.py)

### 검색 계층

```
Layer 1: TOPIC_LAW_MAP  — 키워드 → 기본 법령 세트
Layer 2: keyword_law_map.json — 키워드 → 세부 법령 특정
Layer 3: article_graph.json — 조문 그래프 1-hop 확장
```

### 조문 검색 방식

| 방식 | 설명 | 우선순위 |
|------|------|----------|
| **직접 페칭 (exact)** | law_hints에 명시된 조문을 메타데이터로 직접 쿼리 | 최우선 (score=2.0) |
| **벡터 검색** | 임베딩 유사도 검색 (jhgan/ko-sroberta-multitask) | 중간 |
| **BM25** | 키워드 기반 검색 | 중간 |
| **RRF 병합** | 벡터 + BM25 결합 (벡터 60% / BM25 40%) | — |

### 컨텍스트 구조 (format_context)

```
=== [관련 해석 원칙 메모] ===   ← fetch_linked_memos() 결과 (결정론적 주입)

=== [조문 해석 프레임] ===      ← article_roles JSON (law_hints 기반)

=== [직접 참조 조문] ===        ← exact score 조문 (law_hints 명시 조문)
※ DB에서 직접 가져온 것 — 1차 근거

=== [관련 법령 조문] ===        ← 하이브리드 검색 조문

=== [유사 질의회신 선례] ===    ← qa_precedents + precedents_2026_april

=== [참조 판례 풀] ===          ← court_cases (법규 × 유형 쌍 필터)
```

### 메모 주입 방식 (fetch_linked_memos)

벡터 검색 없이 결정론적으로 매칭:

| 트리거 | 설명 |
|--------|------|
| **linked_to 매칭** | memo.linked_to ∩ 검색된 판례 case_id / 선례 doc_ref |
| **태그 매칭** | memo.tags의 법령+조문 코드 ∈ 검색된 law_docs |

## Pass 2 — 최종 답변 프롬프트

**역할**: Pass 1 분석 결과 + 검색 컨텍스트를 바탕으로 최종 답변 생성.

### 법령 해석 원칙 (6가지)

| # | 원칙 | 핵심 |
|---|------|------|
| 1 | 별표·부속 규정 우선 확인 | 별표 서두·총칙 성격 조문 먼저 검토 |
| 2 | 문언 우선의 원칙 | 문언 명확 시 우선 적용. "등"은 예시 가능 |
| 3 | 엄격 해석의 원칙 | 권리 제한·의무 부과는 좁게. 단, 기계적 적용 금지 |
| 4 | 개념 엄격 분리 | 법령상 별도 정의 개념 혼용 금지 |
| 5 | 특별 규정 우선 | 개별 특별 조문 > 일반 조문 |
| 6 | 형벌법규 연결 확인 | 예외 엄격 해석 시 처벌 범위 확대 역설 발생 여부 검토 |

### 답변 내부 추론 구조 (3트랙)

```
[Step 0] 핵심 용어 법적 정의 확인
    ↓ 정의만으로 해소? → 결론 직행
    ↓ 미해소 → 3트랙으로 진행

① 문언적 해석
    - 중의성 먼저 확인
    - 중의적이면 → "문언상 중의적 — 입법취지·목적론으로 이관"
    - 명확하면 → 문언 주(主) 트랙

② 입법취지 해석
    - 조문 신설/개정 이유·목적 분석
    - 회피 가능성이 결정적 논거인 경우 단정적 결론

③ 목적론적 해석
    - 법 전체 목적 + 실질적 효과 판단
    - 형해화(形骸化) 여부 확인

④ 선례 포지셔닝 (선례 있을 때만)
    - ratio 파악 → 구조 동질성 검증 → 유추 적용 여부 결정
```

### 3트랙 결론 처리

| 상태 | 확신도 | 출력 방식 |
|------|--------|-----------|
| 3트랙 수렴 | **[확정]** | 가장 강한 트랙 완전 전개 + 나머지 1줄 |
| 일부 분기 | **[조건부]** | 다수 트랙 결론 + 소수 이견 명시 |
| 3트랙 분기 | **[해석분기]** | 트랙별 완전 전개 + [해석 분기점] 섹션 |

### 출력 섹션 구조

```
(첫 단락) 결론 선언 — 법제처 회답 형식으로
(이유 단락들) 먼저 ~ / 그리고 ~ / 아울러 ~ / 따라서 ~

### [결론]
[확신도: 확정 | 조건부 | 해석분기]
구체적 조·항 선언

### [관련 조문 확인]
① 직접 적용 조문
② 예외·특례·준용
③ 상충·연관 조문

[관련 판례 검토] (판례 있을 때만)
[근거 법령 + 인용 선례]
[담당부서 확인 질문] (확신도가 확정이 아닐 때만)

[출처 요약]
DB-조문: 참조함 (예: ...) / 참조 없음
DB-선례: 참조함 (예: ...) / 참조 없음
내장지식: 사용함 — (보충 내용 1줄) / 사용 안 함
```

### 출처 우선순위

```
1순위: [직접 참조 조문] — DB에서 직접 페칭한 law_hints 조문
2순위: [관련 법령 조문] — 유사도 검색 조문
3순위: [유사 질의회신 선례] — 관련 있으면 반드시 인용
4순위: 내장지식 — 위에 없을 때만 보충. "(내장지식)" 명시 필수
```
