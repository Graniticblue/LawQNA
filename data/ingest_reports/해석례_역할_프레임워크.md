# 해석례 역할 포지셔닝 프레임워크 (v1, 2026-07-16)

해석례를 주제(논지 계열)로 분류하는 [논지계열_지도.md](논지계열_지도.md)와 별개로,
각 해석례가 시스템 안에서 **어떤 기능적 자리**에 서 있는지를 분류한다.
이 프레임워크는 실측에서 귀납했다: 원류 케이스의 LOO 0점(10-0306·22-0594),
시제 반전 3건, 도메인 횡단 판례의 검색 누락(94누3216) — 전부 "역할을 몰라서
배선과 기대치를 잘못 잡은" 문제였다.

## 축 1 — 논지 수명주기 포지션 (계열 안에서의 자리)

| 코드 | 포지션 | 정의 | 예 |
|---|---|---|---|
| **R** | 원류 (Root) | 논지의 첫 확립 — 계열의 뿌리 | 10-0306(선택 충족), 14-0498('등'), 18-0283(중첩 불가), 22-0594(병존), 25-0141(실질 귀속) |
| **M** | 방법론 (Method) | 논지를 적용하는 절차·판단 순서의 정립 | 18-0018('등' 포섭 3단계), 2007두2555(특별규정 2요건) |
| **A** | 적용·확장 (Application) | 기존 논지를 새 사실·법령에 적용 | 22-0869(10-0306의 사정거리), 21-0143(18-0283의 용적률판), 21-0849(타법령·조례 확장) |
| **B** | 경계 획정 (Boundary) | 논지가 멈추는 곳 — 사정거리의 한계 | 15-0267(요건 초과 시 복귀), 18-0414(이식 불가), 21-0686(호 내부 엄격) |
| **X** | 긴장 케이스 (Cross-tension) | 두 논지가 충돌하는 지점 — 널뛰기 위험 | 21-0142, 12-0126↔16-0506 |
| **L** | 입법 승격 (Legislated) | 정비의견·결론이 입법으로 대체됨 — 개별 결론은 실효, 논지는 승격 | 16-0506(→2019 괄호), 18-0283·21-0849(→2022 60조④) |

한 해석례가 복수 포지션을 가질 수 있다 (21-0849 = A+L, 22-0594 = R(병존)+A('등')).

## 축 2 — 전달 배선 (시스템이 이 논지를 소환하는 경로)

| 배선 | 소환 조건 | 한계 |
|---|---|---|
| 레코드 검색 (기본) | 질의 어휘가 도메인 어휘와 겹칠 때 | 문형·횡단 법리는 어휘 직교로 미소환 (10-0306 실증) |
| 문형 훅 (phrase_principles) | 법정 문형 패턴 감지 시 결정론 주입 | 신호 regex의 리콜 관리 필요 ('등' 훅 결함 실증) |
| 원칙 컬렉션 (P-0XX) | 법리 어휘로 도메인 무관 소환 | 요약 포인터일 뿐 — 본체는 레코드 |
| 조문 프레임 (article_roles) | 해당 조문이 검색되면 동반 | 조문 특정 질의에만 |
| 판례 층 (court_cases) | 법규×유형 페어링 | **타 도메인 판례는 구조적으로 미소환** (94누3216 실증) |

## 배선 규칙 (포지션 → 필수 배선)

- **R(원류)**: 레코드만으로는 부족 — **문형 훅 또는 원칙 컬렉션 또는 프레임에 반드시 배선**.
  원류는 도메인 어휘가 아니라 논지 어휘로 소환돼야 하기 때문 (미배선 시 10-0306형 미소환 재발).
- **M(방법론)**: 프레임 interpretation_logic에 절차를 명문화 + 원류와 연결.
- **A(적용례)**: 레코드 검색만으로 충분 (도메인 어휘가 실려 있음) — 추가 배선 불요.
- **B(경계)**: 해당 계열의 프레임·지도에 caveat로 명기 (경계 없는 논지는 과적용됨).
- **X(긴장)**: 지도에 충돌 쌍으로 등재 — 양쪽 논지를 함께 소환해야 하는 케이스.
- **L(입법 승격)**: **시제 주석 필수**(레코드+프레임) + 후속 입법 조문을 프레임에 등재.
  타 도메인 **판례**의 법리는 판례 층에만 두지 말고 원칙 컬렉션에 병행 배선 (94누3216 규칙).

## 포지션 → eval 기대치 (LOO+컷오프에서 정상 범위)

| 포지션 | 기대 | 벗어나면 의심할 것 |
|---|---|---|
| R | **corr 낮음이 정상** (자기 논지의 선행이 코퍼스에 없음). 단 저점의 원인이 '암묵 전제 공백'이면 해소 가능 — 전제를 독립 출처(조문·형제 해석례)로 명시화해 프레임에 배선하면 LOO에서도 회복된다(17-0651 실증: 3연속 0.0 → 가목 '고시' 개념 앵커 후 1.0, 자기 출처는 LOO 필터로 제거된 상태). 암묵전제_대장 참조 | corr 1.0이면 오히려 leakage 점검 — 회복이 '독립 출처 앵커' 덕인지 자기 결론 누출인지 _strip_role_leakage 작동 확인 |
| M·A | **corr 높아야 정상** (선행 논지가 컷오프 안에 있음) | 낮으면 배선 문제(훅 리콜·검색) 또는 잔여 prior — eval갭 루프 가동 |
| B | 중간 — 원류 있으면 회복 | |
| L | **corr 0.0 = 시제 아티팩트가 정상** | 점수 조작 금지, 주석 확인만 |

→ 오답 진단이 원칙화된다: 포지션을 먼저 확인하고, 기대치 안이면 정상, 밖이면 eval갭 루프.

## 속성 태그 (보조)

- **시제 안정성**: 안정(판례·일반법리) / 주의(입법 후속 존재 — L) / 취약(법령정비의견 부착 — 입법 대기 중, 21-0849가 7주 만에 입법된 전례)
- **컷오프 지지력**: 회신일이 오래될수록 미래 케이스의 LOO 지지대가 됨 (94누3216=1995가 병존 계열 전체의 뿌리인 이유)
- **도메인 결합**: 코어(도메인 어휘 소환 가능) / 횡단(원칙 배선 필수)

## 운영 편입

- 학습 사이클 큐레이션(③) 단계에서 포지션 코드를 정하고, 배선 규칙에 따라 훅·원칙·프레임
  등재 여부를 결정한다. 검수 보고서와 이 문서의 표에 포지션을 기록한다.
- eval(⑥) 판독 시 포지션 기대치를 먼저 대조한다 — R·L의 저점은 정상, M·A의 저점은 루프 가동.
