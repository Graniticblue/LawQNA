# 학습 후보 체크리스트

운영 규칙:
- 후보 발견 시 `- [ ]`로 **등록일·출처**와 함께 추가한다.
- 학습 완료 시 `- [x]`로 바꾸고 **학습일**을 적는다 (검수 보고서는 생성일 폴더 `YYYY-MM-DD/*_학습보고.md`).
- 출처: `리서치`(대화 중 발굴) / `검증큐`(인용 검증 훅이 자동 적재한 실존·미학습 인용 — chroma 볼륨 `_cite_verify/queue.json`) / `인용감지`(학습 코퍼스 내부 인용 스캔 — `python scripts/scan_citation_candidates.py`) / `지정`(사용자 직접 지정) / `eval갭`(eval 오답의 누락 지목 — **최우선 승격**: 오답이 나오면 '이 판례·해석례가 있었으면'을 적극 식별해 우선 학습 후 재평가한다, 2026-07-16 규칙).
- 검증큐의 등장 횟수가 높은 번호부터 우선 승격한다.
- 학습 사이클마다 인용감지 스캔을 돌려 신규 감지분을 이 파일에 반영한다. 도메인 코어는 정식 후보로, 나머지는 맨 아래 '미분류 감지분'에 보존 후 검토·승격한다.
- 후보 선정·학습 완료 시 [논지계열_지도.md](논지계열_지도.md)를 함께 갱신한다 — 후보는 "어느 논지 계열의 빈칸을 메우는가"로 고르고, 완료분은 해당 계열에 등재.
- **라이트트랙(파생·재료 학습, 2026-07-20 규칙)**: 주 대상이 인용한 선행례를 ⑥b(재료 완비 재측정)용으로 학습할 때는 품질 게이트(가드·무결성·시제)는 동일하되 eval만 생략한다. 단 **대장 밖 학습 금지** — 반드시 ① 이 체크리스트에 등재(파생 관계 명기) → ② 학습 완료 표기 → ③ 아래 'eval 후보' 섹션에 측정 대기로 등재하는 3단 장부를 거친다.
