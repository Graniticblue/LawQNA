## 1. 판례-법령 관계 프레임워크 (7가지 유형)

판례가 법령 조문과 맺는 관계를 아래 7가지로 분류하여 라벨링한다.
이 라벨은 검색 시 **필터링 및 가중치 조절**에 활용된다.

### 1.1 유형 정의

| # | 코드 | 이름 | 색상 | 설명 | 예시 |
|---|------|------|------|------|------|
| 1 | `DEF_EXP` | 정의확장형 (Definition Expansion) | 🔵 | 법 조문의 용어 정의를 판례가 넓히거나 구체화 | "대지"의 범위에 사용승인 시 분할예정 토지도 포함 → 건축법 제2조제1항제1호 |
| 2 | `SCOPE_CL` | 적용범위 확정형 (Scope Clarification) | 🟢 | 조문 적용/비적용 경계를 판례가 확정 | 가설건축물에도 건축법 제11조 허가규정 적용 여부 |
| 3 | `REQ_INT` | 요건해석형 (Requirement Interpretation) | 🟡 | 불명확한 요건의 충족 기준을 구체화 | "3층 이상"에 지하층 포함 여부 |
| 4 | `EXCEPT` | 예외인정형 (Exception Recognition) | 🔴 | 문언상 불허이나 특정 조건 하 예외 인정 | 용도변경 없이도 건폐율 완화 가능 |
| 5 | `INTER_ART` | 조문간관계 해석형 (Inter-Article Relationship) | 🟣 | 조문 충돌·중첩 시 우선순위/적용관계 정리 | 건축법 제44조 vs 국토계획법 제56조 적용 우선순위 |
| 6 | `PROC_DISC` | 절차·재량 확인형 (Procedural / Discretion Confirmation) | ⚪ | 허가권자 재량 범위·절차적 요건 확인/제한 | 사용승인 시 분할조건부 복수필지 허가 가능 (재량) |
| 7 | `SANC_SC` | 벌칙·제재 범위형 (Sanction Scope) | 🟤 | 위반 시 제재 범위·이행강제금 기준 구체화 | 이행강제금 부과 시 시정명령 선행 요건 |

### 1.2 라벨 스키마

```json
{
  "relation_type": "DEF_EXP",
  "relation_name": "정의확장형",
  "target_law": "건축법",
  "target_articles": ["제2조제1항제1호"],
  "direction": "expand",
  "summary": "\"대지\"의 범위를 사용승인 시 분할예정 토지까지 확대 해석",
  "confidence": 0.9
}
```

### 1.3 라벨링 전략

**Phase 1 — LLM 기반 자동 라벨링** (초기 구축)
- 판례 텍스트를 Claude에 입력하여 관련 조문 + 관계 유형을 자동 추출
- 프롬프트에 7가지 유형 정의 + few-shot 예시 포함
- confidence ≥ 0.7인 경우만 자동 적용

**Phase 2 — 수동 검증** (품질 확보)
- confidence < 0.7 또는 다중 유형 해당 건 수동 확인
- 샘플 100건 전수 검증 → 라벨링 정확도 측정

**Phase 3 — 검색 시 활용**
- 쿼리 의도에 따라 관계 유형 필터링 (예: "정의가 뭔가요?" → DEF_EXP 우선)
- 관계 유형별 가중치 차등 부여

### 3.3 답변 내 구조 현황 (파싱 가능)

답변은 이미 아래 섹션 구조로 작성되어 있으나, 현재 이를 청킹이나 메타데이터에 활용하지 않고 있다:

```
### [쟁점 식별]
쟁점: ... | 유형: 단일조문형 | 복수조문탐색형 | 조건분기형

### [관련 조문 확인]
① 직접 적용 조문: ...
② 예외·특례·준용: ...
③ 상충·연관 조문: ...

### [사실관계 분기]
...

### [법리적 판단 로직]
1. ...
2. → ...

### [최종 답변]
[확신도: 확정 | 조건부 | 재량위임]
...

### [근거 법령]
- 「법령명」 제○조제○항

### [검색 태그]
#태그1 #태그2
```

**파싱 가능 항목**:
```python
# 섹션 추출 정규식
SECTION_PAT = re.compile(
    r'### \[(?P<name>[^\]]+)\]\n(?P<body>.*?)(?=### \[|\Z)',
    re.DOTALL
)
# 쟁점 유형
TYPE_PAT = re.compile(r'유형:\s*(단일조문형|복수조문탐색형|조건분기형)')
# 확신도
VERDICT_PAT = re.compile(r'\[확신도:\s*(확정|조건부|재량위임)\]')
# 근거 법령
LAW_PAT = re.compile(r'「([^」]+)」\s*(제\d+조(?:의\d+)?(?:제\d+항)?)')
```

## 4. 판례 데이터 파이프라인 (PDF → Chroma)

### 4.1 전체 흐름

```
[판례 PDF 파일]
    │
    ▼
[Step 1] PDF 텍스트 추출 (PyMuPDF / pdfplumber)
    │
    ▼
[Step 2] 판례 구조 파싱 (정규식 기반 섹션 분리)
    │  사건번호, 선고일, 법원, 판시사항, 판결요지, 이유, 주문
    │
    ▼
[Step 3] 법령-판례 관계 라벨링 (LLM 기반)
    │  7가지 유형 자동 분류 + 관련 조문 추출
    │
    ▼
[Step 4] 청킹 → JSONL 저장
    │
    ▼
[Step 5] Chroma 인덱싱 (court_cases 컬렉션)
    │
    ▼
[Step 6] 검색 파이프라인 통합 → 종합계획 참조
```

### 4.2 Step 1 — PDF 텍스트 추출


**추출 전략**:

### 4.4 Step 3 — 법령-판례 관계 라벨링

**모듈**: `13_CaseLabeler.py`

**LLM 라벨링 프롬프트 (핵심)**:

```
당신은 건축법 판례 분석 전문가입니다.
아래 판례의 판결요지/이유를 읽고, 이 판례가 인용하는 각 법령 조문과
어떤 관계를 맺는지 아래 7가지 유형 중에서 분류하세요.

[유형 목록]
1. DEF_EXP (정의확장형): 용어 정의를 넓히거나 구체화
2. SCOPE_CL (적용범위 확정형): 조문 적용/비적용 경계선 확정
3. REQ_INT (요건해석형): 불명확한 요건의 충족 기준 구체화
4. EXCEPT (예외인정형): 문언상 불허이나 예외 인정
5. INTER_ART (조문간관계 해석형): 조문 충돌/중첩 시 우선순위 정리
6. PROC_DISC (절차·재량 확인형): 허가권자 재량/절차 요건 확인·제한
7. SANC_SC (벌칙·제재 범위형): 제재 범위/이행강제금 기준 구체화

[출력 형식 - JSON]
{
  "relations": [
    {
      "relation_type": "DEF_EXP",
      "target_law": "건축법",
      "target_articles": ["제2조제1항제1호"],
      "fact_summary": "분할예정 토지가 건축법상 대지에 해당하는지",
      "logic_summary": "법문의 문언보다 입법 취지를 고려하여 넓게 해석",
      "apply_condition": "사용승인 시 분할이 예정된 경우에 한함",
      "confidence": 0.9
    }
  ]
}

- `fact_summary`: 이 판례가 **뭘** 다뤘는지 (모드 ① 직접 적용 매칭용)
- `logic_summary`: **어떻게** 판단했는지 (모드 ② 논리 차용 매칭용)
- `apply_condition`: 판례 결론의 **적용 조건** (모드 ③ 정의 원용 시 검증용)

[판례 텍스트]
{case_text}
```

**배치 처리**:
- Claude API 호출, 판례 1건당 1회
- Rate limiting: 0.5초 간격
- 결과: `data/case_labels/` 에 JSON 저장
- 실패 건 재시도 큐 관리

## 5. 국가법령 파이프라인 개선 계획

> **현재 파이프라인 상세 구조는 Section 2 (법령 청킹)에 정리되어 있음.**

### 5.1 현재 파이프라인 요약

```
[법제처 OpenAPI] → XML 수집 (01) → JSONL 변환 → Chroma 인덱싱 (02)
[별표 PDF]      → 수동 청킹 (02_B) → JSONL    → 동일 컬렉션에 합류
[질의회신 JSONL] → 직접 인덱싱 (02)  ────────→ qa_precedents 컬렉션
[키워드맵] ← Q&A 태그 자동 추출 (03)
[조문그래프] ← 조문간 참조 파싱 (04)
```

### 5.2 개선 항목

| # | 항목 | 현황 | 개선 방향 |
|---|------|------|----------|
| A | 법령 범위 확장 | 17개 법령 | 건축법 조문에서 「X법」 언급 법률 전체 크롤링 (40~60개) |
| B | 별표 자동 청킹 | 수동 전략 라우팅 | PDF 구조 자동 감지 + LLM 보조 청킹 |
| C | 조문 그래프에 판례 노드 추가 | 조문→조문만 | 조문→판례, 판례→조문 양방향 엣지 |
| D | 키워드맵에 판례 태그 병합 | QA 태그만 | 판례 라벨의 키워드도 매핑 테이블에 반영 |
| E | 임베딩 모델 업그레이드 | ko-sroberta (384d) | 필요 시 multilingual-e5-large (1024d)로 교체 |
| F | 증분 업데이트 | manifest 기반 | 판례 JSONL에도 동일 manifest 패턴 적용 |

### 5.3 조문 그래프 확장 (판례 노드 추가)
